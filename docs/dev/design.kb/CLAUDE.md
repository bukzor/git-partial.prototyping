# Design Decisions

Captures architectural and implementation decisions for git-partial.

## What Belongs Here

- Problem/solution pairs that shaped the design
- Technology choices with rationale
- Constraints discovered during exploration
- Integration patterns and conventions

## What Does NOT Belong

- Implementation details (code)
- User documentation
- Temporary notes or todos

## When to Add

Add a new file when:
- A design question is resolved with clear rationale
- A constraint is discovered that affects architecture
- A technology/approach is chosen over alternatives
